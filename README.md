# LLM-based CV–Vacancy Scoring (MVP)

Прототип AI-модуля для оценки соответствия кандидата вакансии.
На вход подаются тексты вакансии и резюме, на выходе — структурированный и объяснимый скоринг в формате JSON.

Проект реализует базовое AI-ядро, которое может использоваться в системах автоматизированного подбора персонала.

## Ключевая идея

Вместо поиска по ключевым словам или «чёрного ящика» скоринга, пайплайн:
- явно извлекает требования из вакансии,
- проверяет их подтверждение в резюме,
- фиксирует несоответствия и пробелы,
- формирует итоговый скор с объяснением вклада каждого требования.

Это делает результат интерпретируемым и пригодным для использования в HR-процессах.

## Архитектура пайплайна

Пайплайн состоит из двух LLM-шагов:

1. **Извлечение требований**
   - Hard skills, опыт, ограничения, условия.
   - Только то, что явно указано в вакансии.
   - Без добавления предположений.

2. **Скоринг резюме**
   - Каждое требование проверяется отдельно.
   - Статусы `supported` и `contradicted` возможны только при наличии прямой цитаты из резюме.
   - Если подтверждения нет — используется `missing` или `unclear`.

Финальный результат возвращается в строгом JSON-формате.

## Надёжность и объяснимость

- Используется JSON Schema для валидации ответа.
- При невалидном JSON выполняется единичная попытка автоматического исправления без добавления новой информации.
- Для каждого требования сохраняется:
  - цитата из резюме,
  - краткое объяснение,
  - числовой вклад в итоговый скор.

Таким образом, результат можно проверить и понять без знания ML.

## Fairness и работа с данными

Перед скорингом выполняется минимальная санитизация текста:
- маскирование контактных данных и служебной информации;
- игнорирование защищённых признаков (пол, возраст, имя, адрес), даже если они присутствуют в тексте.

Это снижает риск bias и приближает решение к production-подходу.

## Оценка качества

Пайплайн протестирован на выборке из 100 пар «вакансия–резюме» с реальными HR-данными и сильным дисбалансом классов.

Проведена:
- оценка метрик (accuracy / precision / recall / F1) при разных порогах;
- калибровка порога скоринга под разные бизнес-сценарии;
- анализ стабильности JSON-ответов и стоимости пайплайна (latency, tokens).

Итоговый порог скоринга рассматривается как бизнес-настраиваемый параметр, а не фиксированное значение.

## Ограничения и дальнейшее развитие

- Более глубокая предобработка текстов (нормализация, очистка шумных полей).
- Стабилизация и нормализация требований (синонимы, стек технологий).
- Добавление метрик ранжирования (Recall@K, NDCG) для сценария подбора топ-N кандидатов.
- Human-in-the-loop: возможность ручной корректировки требований и весов рекрутером.

## Формат проекта

Проект реализован в одном Jupyter Notebook как self-contained MVP:
- логика пайплайна,
- проверки,
- прогон метрик,
- сохранение артефактов.

Подходит как базовое AI-ядро для дальнейшего расширения и интеграции.
